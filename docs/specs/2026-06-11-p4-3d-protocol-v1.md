## 1. Symmetry scope [D]

The physical symmetry of gravity-aligned tabletop manipulation is **SO(2) about the gravity
axis** (× translations), NOT full SO(3): gravity and the root frame pin the z-axis. The VN
stack is SO(3)-equivariant by construction (stronger than needed — harmless, exact); *claims*
are registered at SO(2)-z scope. Action contract supports this: `pd_ee_delta_pose` translation
deltas are root-frame vectors (G0-banked); the axis-angle rotation part co-rotates under
z-rotations. Full-SO(3) claims would need a frame argument we do NOT make in v1.

- Orbit-transport instrument: `rotate_latent(z, R_z(θ))` with θ ∈ C_N-z grid (N=16 default,
  mirroring the PushT lane); transport tests quantize to the grid (residual ≤ 11.25°, same
  convention as the wedge lane).

- Source: 1000 official demos (motion-planned, `pd_joint_pos`, mean length 149 — G0-banked).

## 4. Training protocol [D, the v1.5 lesson transplanted]

- **Variance floor must respect the representation structure.** PushT v1.5 verdict: per-dim
  std floors fight isotypic anisotropy. The VN analogue is sharper: per-dim variance on
  $\mathrm{vec}(V)$ coordinates is NOT rotation-invariant — a per-dim floor would push the
  latent toward axis-aligned (symmetry-breaking) solutions, recreating the v1.5 pathology by
  construction. Registered recipe: **channel-norm floors** — for vector channels, regularize
  $\mathrm{std}_b(\|V_c - \bar V_c\|)$ (an invariant scalar per channel); for $I$, plain
  per-dim floors. Implementation knob: `var_field_size=3` on the vector block (the existing
  isotypic-floor machinery generalizes — one knob, no new code path) + standard floor on $I$.
- EMA 0.99 start; batch 64; Adam; epochs bound by corpus size (start 20).
- **Health metrics (stage A, before any claim gate):** stability floor = channel-norm std ≥ 0.7
  analogue (exact threshold re-based in G1 — VN scales differ); content probes = tcp xyz $R^2$
  (G0: 0.748 on a 200-step smoke) + peg/hole pose probes (from `extra` obs). Collapse taxonomy
  (stable-but-empty vs collapsed-but-contentful) carries over verbatim.
- **n = 6 runs minimum** per arm for any reported number (v1.6); "runs not seeds" language —
  CUDA determinism flags recorded but not assumed.

## 5. G1 — timescale + conversion measurement (the binding experiment) [registered]

Cheap, box-runnable, BEFORE the first sweep:
1. **G1a conversion validation** (§2 gate) on 20 demos.
2. **Data-only timescales:** per-step tcp displacement norms; per-step cloud chamfer distance;
   medians + dispersion. (No training needed.)
3. **Audit SNR vs stride:** quick VN pairs (n=3, fixed recipe) at strides {1, 2, 5}; audit
   δ̂ growth curves. **Stride binding rule [B-formula]:** smallest stride whose median
   per-chunk tcp displacement ≥ 3× the per-step sensor/render noise floor (estimated from
   static frames), tie-broken toward smaller stride. If inconclusive → CHUNK = 1 + amendment.
4. ε grid for 3D C3 cells: bound as multiples of the G1 δ̂ (same {2,4,8,16}×δ̂ pattern unless
   G1 shows the heavy-tail q90/mean > 2 pathology generalizes — in which case cells are still
   handled by the canonical dual-boundary semantics; `src/audit/gates.py` is mandatory here).

## Amendment A2 (2026-06-12, pre-G2): object-centric symmetry scope + corpus reality

1. **Fixed-base physics:** scene-level rotation is NOT a symmetry of fixed-base manipulation
   (arm kinematics anchor the frame). 3D equivariance claims are scoped **object-centric**:
   the encoder's vector latent co-rotates with OBJECT configuration (peg/hole poses in the
   cloud); robot-arm points are part of the conditioning context, not the symmetry carrier.
   Orbit-transport tests rotate the OBJECT subset / evaluate on naturally-varied object poses
   (PegInsertion randomizes peg/hole poses per episode — natural orbit sampling), never the
   whole scene. Lemma-2 claims inherit this scope.
2. **Corpus (06-12 scope decision):** coverage-grade closed-loop replays (self-consistent by
   construction); goal-directed rows out of scope (G1a-v3 FAIL stands). Stride: CHUNK=1 at
   20 Hz pending the G1.3 SNR binding (registered formula unchanged).
3. **G2 stage-A health:** n = 6; report BOTH per-dim latent std and per-field (size-3) norm
   std; provisional stability floor 0.7 on per-dim std with the VN-specific floor calibration
   EXPLICITLY TBD on first-run distributions (registered as data-bound, not invented).
   Recipe transplant: banked aux0.5_v0.3 family (aux target = tcp xyz), var_field_size=3
   uniform (note: also groups invariant dims in 3s — accepted v0 simplification, disclosed).
